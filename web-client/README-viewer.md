# ScratchJr URL Viewer

A web-based ScratchJr environment that can load `.sjr` project files from the local filesystem or from a URL parameter.

## Requirements

- Node.js 16 or higher
- npm

## Setup (first time only)

After cloning the repository, run:

```bash
node setup.js
```

This will:
1. Install all npm dependencies
2. Build the ScratchJr webpack bundle
3. Apply the necessary patches to make it work in a browser context
4. Copy the bundle to the correct location

This takes about 30–60 seconds.

## Running

```bash
npm start
```

Then open your browser at:

```
http://localhost:8000/editions/free/src/viewer.html
```

## Loading a project

### From the filesystem

Click the **"Open .sjr file"** button in the top-right corner and select a `.sjr` file from your computer.

### From a URL

Add a `file_url` query parameter to the viewer URL:

```
http://localhost:8000/editions/free/src/viewer.html?file_url=https://path/to/file.sjr
```

> **Note:** CORS is only permitted from `localhost:8000` for the test file host. The server must be running on port 8000.

## Project structure

```
web-client/
  web-adapter.js    — Replaces the native iOS/Android tablet interface
  url-loader.js     — Handles file loading from filesystem and URL
  jszip.min.js      — JSZip library for unzipping .sjr files
  settings.json     — ScratchJr settings

editions/free/src/
  viewer.html       — The viewer page
  app.bundle.v2.js  — Patched ScratchJr bundle (generated by setup.js)
  media.json        — Sprite and background library manifest
  settings.json     — Runtime settings

serve.js            — Static file server
setup.js            — One-time setup script
```

## Rebuilding after source changes

If you modify files in `src/`, rebuild and re-patch the bundle:

```bash
node setup.js
```

## Deploying to a cloud server

The app is a plain Node.js static file server — deploy it like any Node.js app.

### Example: deploy to a VPS or cloud VM

```bash
git clone <your-repo>
node setup.js
npm start
```

The server reads `process.env.PORT` so you can set the port via environment variable:

```bash
PORT=3000 npm start
```

### CORS and the file_url parameter

When running on `localhost:8000`, the `file_url` parameter fetches the remote `.sjr` directly from the browser (the test host allows CORS from localhost).

When deployed to any other domain, the fetch is automatically routed through the built-in `/proxy` endpoint on the server. This means `file_url` works on any domain without needing CORS headers on the `.sjr` host.

### Process manager (production)

Use a process manager to keep the server running:

```bash
npm install -g pm2
pm2 start serve.js --name scratchjr-viewer
pm2 save
```

### Reverse proxy (nginx)

If you want to serve on port 80/443 behind nginx:

```nginx
location / {
    proxy_pass http://localhost:8000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
}
```
