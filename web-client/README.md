# ScratchJr Web Client

This directory contains the web-based viewer for ScratchJr projects.

## The Build Process
The core logic of ScratchJr is contained within `app.bundle.js`, which is generated by Webpack from the source code. The build minifies thousands of lines of code into a single file to optimize loading speed.

**Note:** If you are building this bundle yourself, you may encounter an OpenSSL related error. You will need to use the `NODE_OPTIONS=--openssl-legacy-provider` fix when running Webpack.

## The Bridge Logic
To make the native tablet application work in a web browser environment, we use `web-bridge.js`. This script serves as a "Fake Tablet Bridge", mocking the native tablet environment and storage APIs (such as SQLite) so the ScratchJr app can run seamlessly on the web without modifying its core code. 

**Crucially, it is the `window.tablet` object mock that intercepts native calls, which enables browser-based clicks and prevents the UI from hanging.**

## Feature Guide
The web client includes an `auto-loader.js` script that will soon support loading remote projects via a URL parameter.
